# Deploying Finflow to Render

This guide will help you deploy the Finflow application to Render.

## Prerequisites

Before deploying, ensure you have:

1. A **Render account** (sign up at https://render.com)
2. Your code **pushed to GitHub/GitLab** (Render requires a Git repository)
3. The necessary files already created:
   - ✅ `render.yaml` — Infrastructure configuration
   - ✅ `.env.example` — Environment template (in backend folder)
   - ✅ Prisma migrations — Database schema

## Deployment Steps

### 1. Push Your Code to Git

```bash
git add .
git commit -m "prepare for Render deployment"
git push
```

Ensure you push the following important files:

- ✅ `render.yaml` — Service configuration
- ✅ `backend/prisma/migrations/` — Database migrations
- ✅ `.env.example` — Environment template
- ❌ `.env` file itself should NOT be pushed (ignored in .gitignore)
- ❌ `node_modules/` should NOT be pushed (ignored in .gitignore)
- ❌ `dist/` folders should NOT be pushed (ignored in .gitignore)

### 2. Deploy on Render

#### Option A: Using Infrastructure as Code (render.yaml) ⭐ Recommended

1. Go to https://dashboard.render.com
2. Click **New** → **Blueprint**
3. Connect your Git repository
4. Select your repository and branch
5. Render will automatically detect `render.yaml` and configure your services
6. Review the environment variables and click **Deploy**

**Environment Variables Set Automatically:**

- `NODE_ENV=production`
- `PORT=3000`
- `DATABASE_URL=file:./finance.db`
- `JWT_SECRET` — Generated randomly by Render

#### Option B: Manual Web Service Setup

If you prefer not to use `render.yaml`, create services manually:

1. **Backend Service:**
   - Go to https://dashboard.render.com → New → Web Service
   - Connect to your repository
   - Build Command: `npm run install:all && npm run build`
   - Start Command: `npm run db:migrate:deploy && npm start`
   - Environment Variables:
     ```
     NODE_ENV=production
     JWT_SECRET=[generate a secure random string]
     DATABASE_URL=file:./finance.db
     ```

### 3. Access Your Deployed Application

After deployment completes:

- **Frontend:** `https://your-service-name.onrender.com`
- **API Docs:** `https://your-service-name.onrender.com/api-docs`
- **API Health:** `https://your-service-name.onrender.com/api/health`

### 4. Test the Deployment

1. Visit your frontend URL
2. Log in with demo credentials:
   - **Admin:** admin@finflow.com / admin123
   - **Analyst:** analyst@finflow.com / analyst123
   - **Viewer:** viewer@finflow.com / viewer123

## Important Notes

### Database Persistence with SQLite

⚠️ **Note on SQLite:** Render has ephemeral storage, meaning your SQLite database will be reset when your service redeploys or goes offline. This is fine for development/testing, but for **production data persistence**, consider:

**To upgrade to PostgreSQL:**

1. Add a PostgreSQL database service to `render.yaml`
2. Update `DATABASE_URL` to the PostgreSQL connection string
3. Update `backend/prisma/schema.prisma` provider from `sqlite` to `postgresql`
4. Push changes and Render will redeploy automatically

### JWT Secret

The `JWT_SECRET` is auto-generated by Render. To use a specific secret:

1. Go to your service settings on Render dashboard
2. Edit the `JWT_SECRET` environment variable
3. Save and trigger a redeploy

### Logs

Monitor your deployment:

1. Go to your service on Render dashboard
2. Click **Logs** to view real-time output
3. Check for any build or runtime errors

## Troubleshooting

### Build Fails with "Cannot find module"

- Ensure all dependencies are listed in `package.json` and `backend/package.json`
- Try deleting `node_modules/` locally and running `npm install:all` again

### Database Migration Fails

- The migration is applied during startup via `db:migrate:deploy`
- Check logs for specific Prisma errors
- Ensure `backend/prisma/migrations/` folder is pushed to Git

### Frontend Not Loading

- Verify TypeScript compilation succeeded
- Check browser console for API endpoint errors
- Ensure environment variables are set correctly

### API Requests Return 404

- The backend should be serving both API routes and static frontend files
- Check that frontend build output is in `frontend/dist/`

## Updating Your Deployment

To redeploy after code changes:

```bash
git add .
git commit -m "your changes"
git push
```

Render will automatically detect the push and redeploy your service.

**To manually trigger a redeploy:**

1. Go to service settings on Render dashboard
2. Click **Manual Deploy** → **Deploy latest commit**

## Production Recommendations

For a production deployment, consider:

1. **Use PostgreSQL instead of SQLite** — For data persistence
2. **Enable HTTPS** — Automatically provided by Render
3. **Set up environment variables securely** — Use Render's environment variable UI
4. **Monitor logs** — Check Render logs for errors
5. **Set up alerts** — Configure notifications for deployment failures
6. **Enable auto-deploys** — Already configured via render.yaml
